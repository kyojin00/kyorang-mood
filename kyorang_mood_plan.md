# 교랑무드 (Kyorang Mood) — MVP 기획 문서

## 1. 제품 개요

- **이름**: 교랑무드 (`kyorang_mood`)
- **소속**: 교랑(Kyorang) 브랜드 패밀리 — 교랑톡의 형제 앱
- **한 줄 정의**: 내 기분을 기록하고 관리하면, 거기에 맞춰 필요한 것을 챙겨주는 감정 중심 앱
- **중심**: 기분(감정관리)이 메인, 비서 기능이 보조
- **타깃**: 국내 대중 시장
- **수익 모델(예정)**: 무료 큐레이션 + 유료 구독(AI 생성 제안 등)

핵심 진입점은 "오늘 내 기분이 어때"이고, 앱이 거기에 반응해 제안을 건넨다.
감정이라는 명확한 입구가 있어 일반 비서 앱과 차별화된다.

---

## 2. 브랜드 / 테마

- 교랑 패밀리 공통 다크 퍼플 톤
  - 배경 `#060610`, 액센트 `#7C3AED`
- 기분 단계별 색상 팔레트 (어두운 보라 → 밝은 보라)
  - 1 아주 나쁨 / 2 나쁨 / 3 보통 / 4 좋음 / 5 아주 좋음
- 차분하고 따뜻한 정서, UI 이모지 최소화
- 색상은 하드코딩 금지, 전부 `AppTheme`를 통해 사용

---

## 3. 핵심 기능 (MVP)

1. **기분 기록** — 탭 한 번으로 기분(1~5) 선택 + 선택적 한 줄 메모
2. **맞춤 제안** — 성향 + 기분에 맞는 큐레이션 문구 1개 + "도움 여부" 반응
3. **돌아보기** — 월 달력에 기분을 색으로 표시, 날짜별 상세
4. **통계** — 월간 요약 / 기분 추이 선그래프 / 기분 분포
5. **온보딩** — 질문 3개로 성향 분류 + 선호 시간대 파악

---

## 4. 성향 시스템

성향 4유형은 제안 유형(`SuggestionType`)과 1:1 대응한다.
"사용자 성향 = 주로 건넬 제안 유형".

| 성향 | 설명 | 제안 콘텐츠 |
|------|------|------------|
| 글귀형 (quote) | 위로/공감 문장을 원함 | 위로·공감 글귀 |
| 휴식형 (rest) | 멈춤/혼자만의 시간 | 호흡·이완·멈춤 제안 |
| 활동형 (activity) | 몸을 움직여 풀림 | 작은 행동 제안 |
| 연결형 (connect) | 누군가와 연결될 때 회복 | 안부·연결 권유 (→ 교랑톡 연동 지점) |

### 온보딩 분류 로직
- 질문 1 (주 신호): 답 → 해당 성향 +2
- 질문 2 (보조 신호): 답 → 관련 성향 +1
- 질문 3: 선호 시간대 (분류 무관, 추후 비서 알림용)
- 점수 합산 후 최고점 성향, 동점 시 질문 1 우선

---

## 5. 제안 시스템

- 방식: **직접 큐레이션** (MVP). 서버·AI 없음.
- 구조: 성향 4 × 기분 구간 3 = 12 카테고리, 각 3문구 (총 36개)
  - 기분 구간: low(1~2) 위로 / mid(3) 환기 / high(4~5) 누림
- 반응(`suggestionHelpful`)을 저장 → 추후 추천 보정 데이터로 활용

---

## 6. 화면 구조

```
온보딩 (최초 1회)
   └─ 질문 3개 → 성향 분류 → 저장 → 홈으로 자동 전환
홈 (기분 기록)
   ├─ 기분 선택 → 메모 → 기록하기
   ├─ 기록 후: 기분 확인 + 맞춤 제안 카드
   └─ 달력 아이콘 → 돌아보기
돌아보기
   ├─ 월 달력 (기분 색 표시) + 날짜별 상세
   └─ 통계 아이콘 → 통계
통계
   └─ 월 요약 / 기분 추이 그래프 / 기분 분포
```

`main.dart`의 `_RootScreen`이 온보딩 완료 여부를 보고 온보딩 ↔ 홈을 분기한다.

## 7. 기술 스택

- **프레임워크**: Flutter (Dart 3.11)
- **상태관리**: Riverpod 3.x (`Notifier` 패턴)
- **로컬 저장**: hive_ce / hive_ce_flutter (JSON 문자열 저장)
- **차트/달력**: 외부 패키지 없이 `CustomPainter`로 직접 구현
- 서버(Supabase) 동기화는 미적용 — 모델에 `toJson`/`fromJson` 준비됨

## 9. 데이터 모델

### MoodEntry (기분 기록)
- `id`, `date`, `moodLevel`(1~5)
- `note`(선택), `suggestion`(선택), `suggestionType`(선택)
- `suggestionHelpful`(반응, 추후 보정용)

### UserProfile (사용자 프로필)
- `persona`(SuggestionType), `preferredTime`(아침/저녁/그때그때)
- `onboardingDone`

---

## 10. 로드맵 (검증 후 단계적 추가)

1. **추천 보정** — `suggestionHelpful` 데이터로 제안 정교화
2. **AI 생성 제안 + 구독** — 무료=큐레이션 / 유료=AI 맞춤 메시지
3. **비서 기능 강화** — `preferredTime` 기반 알림·아침 브리핑(제로클릭)
4. **교랑톡 연동** — 연결형 제안 → 교랑톡 안부 전하기, 계정 통합
5. **음악·외부 연동** — 라이선스·기술 부담 커서 마지막

---

## 11. 핵심 원칙 / 결정사항

- 1인 기준이 아니라 **키울 사업**으로 접근 (대중 시장, 기술 차별화)
- MVP는 **가장 가볍게** 만들어 출시 → 검증 후 무거운 것 추가
- 새 아이디어보다 **검증**이 우선
- 출시 전 **이름 검증** 필요: 키프리스 상표 / 앱스토어 / 도메인 / SNS
  - "교랑" 모브랜드 자체를 상표 등록하면 패밀리 전체 보호
